# デバッグ用VRMビューア 要求仕様書

## 背景

talava-trackerはOSC経由でVMTにトラッカーデータを送信し、SteamVR上のNalulunaAvatars（Beat Saber Mod）でVRMアバターを駆動している。しかしトラッキングの問題（ジャンプ、ドリフト、座標ズレ等）を調査する際、SteamVR + Beat Saberを起動して実機確認するしか手段がない。

トラッカーデータを直接可視化し、VRMモデルで動作確認できるデバッグツールが必要。

### 現状

Beat Saber内ではNalulunaAvatars（非OSS Mod）でVRMアバターを駆動しているが、参考実装としては使えない。

## 検討経緯

### 技術選定

以下の選択肢を比較検討した。

| 選択肢 | VRM対応 | IK | クロスプラットフォーム | 開発コスト |
|---------|---------|-----|----------------------|-----------|
| Three.js + three-vrm | 成熟（Pixiv公式） | 自前 | ブラウザで完全 | 中 |
| Godot + VRM addon | アドオン | 組み込み | ビルド必要 | 中 |
| Unity + UniVRM | 最も成熟 | FinalIK等 | ビルド必要 | 小〜中 |
| Bevy (Rust) | 未成熟 | 自前 | ビルド必要 | 大 |

### 決定: Web（Three.js + three-vrm）

理由:

- **クロスプラットフォーム**: ブラウザさえあればMac/Windows両方で動く。ビルド不要
- **Mac単体デバッグ**: 録画データの再生だけでMac上で分析可能
- **VRM対応**: three-vrmはPixiv公式メンテで成熟
- **デバッグUI**: HTML/CSSで柔軟にUIを構築可能（グラフ、数値表示、設定パネル）
- **独立性**: talava-tracker本体（Rust）を汚さず別プロジェクトとして開発可能

### パフォーマンス評価

| 項目 | 評価 | 根拠 |
|------|------|------|
| VRM 1体の描画 | 問題なし | 最小シーン構成。ブラウザVRM追従アプリの実績あり（Kalidoface 3D, VRoom等） |
| IK計算 | 問題なし | 6点チェーンIKはJS/TSで十分軽量 |
| 90Hzデータ受信 | 問題なし | WebSocket受信はrequestAnimationFrameと独立 |
| WebSocket遅延 | 許容（30-70ms） | ローカル通信。デバッグ用途なら問題にならない |
| 描画FPS | モニタ依存 | requestAnimationFrameはモニタのリフレッシュレートに従う |

## アーキテクチャ

```
[inference_server] --OSC/UDP--> [bridge server (Node/Deno)]
                                    |
                                    | WebSocket
                                    v
                                [ブラウザ ビューア]
                                 ├── three-vrm (VRM描画)
                                 ├── IK (トラッカー→ボーン変換)
                                 ├── スケルトン/マーカー描画
                                 ├── テレメトリUI
                                 └── 録画/再生
```

### inference_server側の変更

VMTへの送信と並行して、デバッグビューア向けにOSCを複製送信する機能を追加する。送信先アドレス・ポートはinference_server.tomlで設定可能にする。

### bridgeサーバー

OSC (UDP) を受信し、WebSocketでブラウザに中継する軽量プロセス。

### ブラウザビューア

静的ファイル配信 + WebSocket接続。別プロジェクトとして開発する。

## 入力データ仕様

inference_serverが送信する既存のVMTプロトコルをそのまま使用する。

OSCアドレス: `/VMT/Room/Unity`

```
args: [index: i32, enable: i32, timeoffset: f32, x: f32, y: f32, z: f32, qx: f32, qy: f32, qz: f32, qw: f32]
```

| index | 部位 | 位置の元データ |
|-------|------|---------------|
| 0 | 腰 | Hip中点 |
| 1 | 左足 | LeftAnkle |
| 2 | 右足 | RightAnkle |
| 3 | 胸 | 肩-腰内挿 |
| 4 | 左膝 | LeftKnee |
| 5 | 右膝 | RightKnee |

- 位置: メートル単位
- 回転: クォータニオン [qx, qy, qz, qw]。基本的にYaw（Y軸回転）のみ
- 送信レート: 90Hz
- enable: 0=無効, 1=通常トラッカー

座標系:

| 軸 | 意味 |
|----|------|
| x | 左右 |
| y | 上下 |
| z | 前後（奥行き） |

## 機能要件

### F1. OSCデータ受信

- bridgeサーバー経由でWebSocketで `/VMT/Room/Unity` メッセージを受信する
- 受信ポートは設定で変更可能
- 90Hzの全フレームを取りこぼさず処理する

### F2. 録画・再生

- 受信したトラッカーデータをタイムスタンプ付きで保存する（JSONLフォーマット）
- 録画ファイルをブラウザで読み込み、再生できる
- 操作: 再生/一時停止、コマ送り（前後）、再生速度変更（0.25x / 0.5x / 1x / 2x）
- Mac単体でのオフライン分析を主要ユースケースとする

### F3. 3Dビューア

- 床グリッド、座標軸（XYZ色分け）を描画する
- カメラ操作: オービット回転、ズーム、パン
- 描画FPSはモニタのリフレッシュレートに従う（requestAnimationFrame）

### F4. トラッカー可視化

- 6トラッカーを色分けした球体マーカーで描画する
- 各トラッカーのローカル座標軸をギズモ（矢印）で描画する
- トラッカー間をスケルトン線で接続する（腰→膝→足、腰→胸）
- 各トラッカーの位置(xyz) / 回転(qxyzw)を数値でオーバーレイ表示する
- enable=0のトラッカーは半透明または非表示にする

### F5. VRMモデル表示

- VRMファイル（0.x）を読み込み描画する
- 6トラッカー位置からIKでVRMボーンを駆動する
  - Hips: 腰トラッカー位置を直接適用
  - Spine/Chest: 腰→胸方向からSpine回転を算出
  - 下半身: 腰→膝→足の3点チェーンでLeg IK
  - 腕: データなし。デフォルトポーズ固定
  - 頭: データなし。デフォルトポーズ固定
- 表示モード切替: VRMモデル ON/OFF、半透明モード（スケルトンとの重畳表示）

### F6. テレメトリ

- OSCデータの実測受信レート（Hz）を表示する
- 選択したトラッカーのx/y/z座標を時系列折れ線グラフで表示する（直近数秒分）
- フレーム間の位置変化が閾値を超えたとき、マーカー色を変えてハイライトする

### F7. 設定

- OSC受信ポート
- VRMファイルパス
- 設定はブラウザのlocalStorageに永続化する

## スコープ外

- NalulunaAvatarsの代替（本番VRMアバター描画）
- VMCプロトコル対応
- VRM 1.0対応（必要になった時点で追加）
- 音声・リップシンク
- 複数人トラッキング
- リモートデバッグ（LAN越しの接続。ローカル通信を前提とする）
