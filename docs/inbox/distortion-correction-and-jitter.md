# 歪み補正とジッタの関係

## 問題の構造

推論モデルのキーポイント検出は毎フレーム±数pxブレる（不可避）。
歪み補正なしだと、このブレが歪み関数で**非線形に増幅**される。

- 画像端ほど増幅が大きい
- カメラごとに歪み特性が異なるため、同じブレでも3D誤差の方向・大きさがバラバラ
- 三角測量で統合すると3D座標がフレームごとに不規則に跳ねる → ジッタ

## 現状

- キャリブレーションで歪み係数は**求めている**が、三角測量時に**使っていない**
- コード上で全カメラの`dist_coeffs`が`[0; 5]`で上書きされている
- cam2はk2=11.5, k3=-49.9と極端 → 数百〜数千pxの誤差

## 解決策

三角測量前に2D観測座標を歪み補正（undistort）する。
2D座標と射影行列の座標系が一致し、三角測量精度が向上する。
ノイズが減ればOne Euro Filterを弱く/なくせる → 動き出し遅延も解消。

## フィルタの限界（One Euro Filter）

- min_cutoffを上げる → 静止時ジッタが残る
- min_cutoffを下げる → 動き出しが遅れる
- betaを上げても「速度検出→カットオフ上昇」に数フレームかかる構造的限界
- フィルタなしなら遅延ゼロだが、現状はノイズが大きすぎて外せない
- **根本解決は歪み補正の有効化**
